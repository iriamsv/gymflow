# Agile, Scrum y Kanban

## ¿Qué es Agile?

Agile es una metodología de desarrollo de software basada en la entrega continua de valor mediante ciclos cortos de trabajo. Su objetivo es adaptarse rápidamente a los cambios y mejorar la colaboración entre los miembros del equipo y los clientes.

## ¿Qué es Scrum?

Scrum es un marco de trabajo Agile que organiza el desarrollo en ciclos llamados sprints, normalmente de entre una y cuatro semanas.

### Roles principales

* Product Owner: define los requisitos y prioridades.
* Scrum Master: ayuda al equipo a seguir Scrum.
* Development Team: desarrolla el producto.

### Conceptos importantes

* Product Backlog
* Sprint Backlog
* Sprint
* Sprint Review
* Sprint Retrospective

## ¿Qué es Kanban?

Kanban es un método visual para organizar tareas mediante columnas que representan estados de trabajo.

Ejemplo:

* Backlog
* Todo
* In Progress
* Review
* Done

## Diferencias entre Scrum y Kanban

Scrum trabaja mediante sprints con duración fija y roles definidos.

Kanban utiliza flujo continuo de trabajo y es más flexible respecto a cambios y prioridades.

## ¿Cuándo usar cada uno?

Scrum es recomendable cuando existe un equipo estable y objetivos definidos para cada iteración.

Kanban es útil cuando el trabajo llega de forma continua y se necesita flexibilidad para reorganizar prioridades.
